## Bindende Konventionen (Stand jetzt)

- **Design-System ist verbindlich (NEU seit M13):** alle UI folgt `docs/design.md` (Signalpult) — semantische Token-Utilities statt Hex, Space Grotesk + JetBrains Mono (Mono nur für Maschinendaten: Server-IP/Port/Slots/RAM, Scores, IDs), ein rationierter Amber-Akzent, `LiveIndicator` für live/running, alle vier Zustände. **Für jede UI-Arbeit zuerst den `frontend-design`-Skill aufrufen.** Verankert in `CLAUDE.md` (Conventions) + einem **PreToolUse-Hook** (`.claude/settings.json` + `.claude/hooks/design-reminder.sh`) auf `.vue`/`.css`/`resources`-Edits. *Hook-Hinweis: er greift erst nach einmaligem `/hooks` (Config-Reload) oder Neustart, falls die Session vor Anlegen der settings.json startete — in einer frischen Session ist er automatisch aktiv.*
- Privilegien-/State-Felder nie `$fillable` (nur via Actions/Jobs; Factories umgehen fillable → Tests nutzen Factory).
- Jede Autorisierung über Policy; nie Client-IDs (`$request->user()`; Discord via `discord_id`). **Helfer-Rolle** (`isHelper()`, M5) bedient Trigger/Server-Ops, wo die Roadmap „Orga/Helfer" sagt; `canAccessPanel()` bleibt `isOrga()`.
- Input-Validierung in die Domain-Action (nicht nur FormRequest). Actions: `DB::transaction` + Parent-Row-`lockForUpdate()` zuerst; 23505-Narrowing.
- Typisiertes jsonb über Cast+DTO (Muster `MenuCast`/`SceneConfigCast`), nie rohes Filament-`KeyValue`.
- Externe Systeme nur über Contracts + Fakes, nie echt in Tests; `Http::preventStrayRequests()` global — neue betroffene Testdirs ergänzen (`Feature/GameServers`, `Unit/GameServers`).
- Provisioning: thin `ShouldQueue`-Listener → `ShouldQueue`-Job (idempotent, nach Commit, blockt nie die Bracket-Progression-Transaktion); delayed Cleanup.
- Neue Modul-Filament-Resources in `AdminPanelProvider` per `->discoverResources(in:, for:)`; Modul-Commands in `bootstrap/app.php`/`routes/console.php`.
- Commits: Conventional Commits, **nur task-spezifische Pfade stagen**, Trailer `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`. Zwischendurch zu `origin` pushen; Roadmap-Doc + GitHub-Milestones/Board laufend aktuell halten.

## SDD-Mechanik (kurz)

- Ledger `.superpowers/sdd/progress.md` (repo-root, gitignored) = Wahrheit über Fortschritt; nach Compaction Ledger + `git log` vertrauen, nicht der Erinnerung.
- Skripte unter dem `subagent-driven-development`-Skill-Dir: `scripts/task-brief PLAN N` (Brief) + `scripts/review-package BASE HEAD` (Diff-Package). BASE pro Task = HEAD vor dem Implementer (nie `HEAD~1`).
- Modelle explizit: Implementer/Reviewer/Fix = **sonnet**, Whole-Branch-Review = **opus**.
- **Agent-Falle:** Implementer idlen/spawnen manchmal Kinder statt selbst zu bauen → immer „do ALL work yourself, do NOT spawn subagents or wait" dispatchen.
- **Task-Zählung gegen den Plan prüfen** (bei M5 wurde T14 fast übersehen — der Plan hat 13 Tasks, T13 = Infra/Spike).
