## Bindende Konventionen (aus allen bisherigen Reviews)

- Privilegientragende/State-Felder **nie `$fillable`** (nur via Actions/`forceFill`; Factories umgehen fillable → Tests nutzen Factory statt `create()`).
- **Jede Autorisierung über Policy**; nie Client-gelieferte User-IDs (Discord-User via `discord_id`; Web-User via `$request->user()`).
- **Input-Validierung in die Domain-Action**, nicht nur in den `FormRequest` (Slash-Commands/Nicht-HTTP-Aufrufer umgehen den FormRequest sonst).
- Actions: `DB::transaction` + **Parent-Row-`lockForUpdate()` zuerst** für Kapazitäts-/Fenster-/Transition-Checks; `QueryException`-Catch auf SQLSTATE `23505` verengen.
- **Typisiertes jsonb** über Cast/DTO (nicht rohes Filament-`KeyValue`); Filament-Formular mit typisierten Feldern.
- **Neue Modul-Filament-Resources** in `AdminPanelProvider` per `->discoverResources(in:, for:)` registrieren; Modul-Console-Commands in `bootstrap/app.php` `withCommands([...])`.
- **Broadcasting** öffentlich auf `event.{id}` (in `routes/channels.php` dokumentieren, keine private Daten im Payload); Frontend via `useEventChannel`/`useTournamentChannel`-Muster.
- **i18n-Gate:** je lang/de-Task mindestens eine Label-Assertion (`->where('labels.x', 'Deutsch')`); `.env.testing` hat `APP_LOCALE=de`.
- Externe Systeme nur über Contracts + Fakes, nie echt in Tests (`Http::preventStrayRequests()` global in Discord/Voice/Lfg-Testdirs — neue betroffene Testdirs ergänzen).
- Commits: Conventional Commits, **nur task-spezifische Pfade stagen** (nie `git add -A`), Trailer `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
