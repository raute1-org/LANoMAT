# LANoMAT v2 — M3 Teams, Turniere, Discord-Interactions, Mumble: Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. **Reihenfolge ist verbindlich:** Die Bracket-Engine (Tasks 4–7) ist reine Domain-Logik ohne IO und MUSS vollständig grün sein, bevor irgendein Lifecycle-/UI-Task (ab Task 8) sie konsumiert.

**Goal:** Ein Turnier läuft komplett digital: Teams bilden → Turnier-Enrollment (solo/Team) → Check-in-Fenster → Auto-Start → Bracket wird generiert und live via Reverb angezeigt → Teilnehmer melden Ergebnisse, Gegner bestätigt (Dispute bei Konflikt) → `BracketProgressor` schiebt Sieger/Verlierer weiter → genau ein Sieger. Match-Koordination läuft über automatisch erstellte Discord-Text-Channels und Mumble-Voice-Channels; Slash-Commands (`/tournament`, `/help`) spiegeln die Web-Actions über einen Ed25519-verifizierten Interactions-Endpoint.

**Architecture:** Ein neues Kernmodul plus zwei Erweiterungen bestehender Module im modularen Monolithen (Design-Dokument Abschnitt 6/11):
- `app/Modules/Teams/` — `Team` + `TeamMember` + `TeamJoinRequest`-Models, Actions (`CreateTeam`, `InviteToTeam`/`RequestToJoin`, `RespondToJoinRequest`, `LeaveTeam`), `TeamPolicy`, Filament `TeamResource`, Inertia-Seiten.
- `app/Modules/Tournaments/` — **Herzstück.** `Domain/` (reine Bracket-Engine: `Slot`, `BracketMatch`, `BracketPlan`, `BracketGenerator`, `BracketProgressor` — kein IO, exhaustiv Pest-getestet), Models (`Tournament`, `TournamentEntry`, `Match`, `MatchReport`), Actions (Enrollment, Check-in, `StartTournament`, Ergebnis-Flow), Domain-Events (`TournamentStarted`, `MatchReady`, `MatchCompleted`, `TournamentCompleted`, broadcast auf `tournament.{id}`), Filament `TournamentResource`, Inertia Bracket-UI.
- `app/Modules/Discord/` (Erweiterung) — Interactions-Endpoint mit Ed25519-Middleware, Command-Router, `discord:register-commands`, Match-Channel-Jobs (`CreateMatchChannelJob`/`CleanupMatchChannelJob`) als Listener auf `MatchReady`/`MatchCompleted`. `HttpDiscordClient` bekommt `Http::retry` + 429-Handling; Sends werden gequeued (M2-Review).
- `app/Modules/Voice/` — `MumbleClient`-Contract + `HttpMumbleClient` + `FakeMumbleClient`, Voice-Orchestrierung (Channel-Baum je Turnier, temporäre Match-Channels), Join-Link-Helper.

> **Modul-Entscheidung (verbindlich, mit Begründung):** `Tournaments` ist ein eigenständiges Top-Level-Modul mit `Event` als organisatorischer Aggregatwurzel (`tournaments.event_id`). Teams sind **cross-event** (kein `event_id`) — sie sind globale Stamm-Entitäten, deren Roster beim Turnier-Enrollment als `roster_snapshot` eingefroren wird (Design Abschnitt 6.5/6.6). Voice ist ein Querschnittsmodul analog zu Discord (eigenes Contract-Interface, mockbar). Die Bracket-Engine lebt bewusst in `Tournaments/Domain/` und darf **keine** Eloquent-/Facade-/IO-Abhängigkeit haben — sie ist reine PHP-Domänenlogik.

**Tech Stack:** PHP 8.4, Laravel 13, Filament v5 (5.6.8), Inertia v2, Vue 3, Tailwind v4, shadcn-vue, **Laravel Reverb + Echo (neu in M3)**, Pest 4, Larastan Level 8, Pint, PostgreSQL 16 (Host-Port 5434), Redis (Host-Port 6380), **Mumble (mumble-server + Ice-REST-Sidecar, neu in M3)**, `ext-sodium` (Ed25519-Signaturprüfung, PHP-Core).

namespace App\Modules\Tournaments\Domain;

final readonly class Slot

    }

    public function isBye(): bool { return $this->bye; }
    public function isPending(): bool { return $this->pendingFromMatchId !== null; }
    public function isEmpty(): bool { return ! $this->isEntry() && ! $this->isBye() && ! $this->isPending(); }
    public function entryId(): ?int { return $this->entryId; }
    public function pendingFromMatchId(): ?int { return $this->pendingFromMatchId; }
}

Über diese Sequenz werden `entryIds` (1-indexiert Seed) den Runde-1-Slots zugeordnet; fehlende Plätze (size > n) sind `bye`.

use App\Modules\Tournaments\Domain\Bracket;
use App\Modules\Tournaments\Domain\BracketGenerator;
use App\Modules\Tournaments\Domain\BracketProgressor;

function playRandomly(\App\Modules\Tournaments\Domain\BracketPlan $plan, int $seed): \App\Modules\Tournaments\Domain\BracketPlan
{
    $rng = new \Random\Randomizer(new \Random\Engine\Mt19937($seed));
    $progressor = app(BracketProgressor::class);

    // Repeatedly find a playable match (both real slots filled, not yet decided) and apply a random result.
    do {
        $playable = collect($plan->matches())->first(fn ($m) => $m->isPlayable() && ! $m->isDecided());
        if ($playable === null) { break; }
        $s1 = $rng->getInt(0, 2); $s2 = $s1 === 2 ? $rng->getInt(0, 1) : 2; // ensure a winner, no draw
        $plan = $progressor->apply($plan, $playable->id, $s1, $s2);
    } while (true);

    return $plan;
}

it('yields exactly one champion and no open matches for random single-elim results', function (int $n) {
    foreach (range(1, 5) as $seed) {

        $open = collect($plan->matches())->filter(fn ($m) => $m->isPlayable() && ! $m->isDecided());
        expect($open)->toBeEmpty();
        expect($plan->finalMatch()->winnerEntryId())->not->toBeNull();
    }
})->with([2, 3, 5, 8, 13, 16, 32]);

it('yields exactly one champion for random double-elim results', function (int $n) {
    foreach (range(1, 5) as $seed) {

        $open = collect($plan->matches())->filter(fn ($m) => $m->isPlayable() && ! $m->isDecided());

        $champion = collect($plan->matches())
            ->filter(fn ($m) => $m->bracket === Bracket::Finals && $m->isDecided())

        expect($champion?->winnerEntryId())->not->toBeNull();

})->with([4, 8, 16]);

```php
<?php

use Illuminate\Support\Facades\Config;

function signedInteraction(array $body): array
{
    $keypair = sodium_crypto_sign_keypair();
    $secret = sodium_crypto_sign_secretkey($keypair);
    $public = sodium_crypto_sign_publickey($keypair);
    Config::set('services.discord.public_key', bin2hex($public));

    $json = json_encode($body);
    $timestamp = (string) time();
    $sig = bin2hex(sodium_crypto_sign_detached($timestamp.$json, $secret));

    return [$json, $timestamp, $sig];
}

it('rejects an invalid signature', function () {
    [$json, $timestamp] = signedInteraction(['type' => 1]);

    $this->call('POST', '/api/discord/interactions', [], [], [],
        ['HTTP_X-Signature-Ed25519' => str_repeat('0', 128), 'HTTP_X-Signature-Timestamp' => $timestamp,
         'CONTENT_TYPE' => 'application/json'], $json)
        ->assertStatus(401);
});

it('answers PING with PONG for a valid signature', function () {
    [$json, $timestamp, $sig] = signedInteraction(['type' => 1]);

    $this->call('POST', '/api/discord/interactions', [], [], [],
        ['HTTP_X-Signature-Ed25519' => $sig, 'HTTP_X-Signature-Timestamp' => $timestamp,
         'CONTENT_TYPE' => 'application/json'], $json)
        ->assertOk()
        ->assertJson(['type' => 1]);
});
